Charts

Pie chart

Configuration

Any parameter with a value column can be displayed as a pie chart.

Example: Model Pickstock, parameter weight

JSON example


{
  "dataRendering": {
    "stock_weight": {
      "outType": "graph",
      "graph": {
        "title": "weight",
        "tool": "plotly",
        "labels": "symbol",
        "values": "value",
        "type": "pie"
      },
      "height": 700
    }
  }
}

Donut chart

Configuration

A donut chart is a pie chart with a hole. In the configuration mode the donut chart is therefore to be found under pie chart. Only the size of the donut hole can be configured here additionally.

Example: Model Pickstock, parameter weight

JSON example


{
  "dataRendering": {
    "stock_weight": {
      "outType": "graph",
      "graph": {
        "title": "weight",
        "tool": "plotly",
        "labels": "symbol",
        "values": "value",
        "type": "pie",
        "hole": 0.6
      },
      "height": 700
    }
  }
}

Bar chart

Configuration

If a parameter is to be visualized as a bar chart, an index must be specified for the x-axis as well as for the y-axis. Several data series can be plotted simultaneously on the x-axis.

Example: Model Pickstock, parameter weight

JSON example


{
  "dataRendering": {
    "stock_weight": {
      "outType": "graph",
      "graph": {
        "title": "weight",
        "tool": "plotly",
        "type": "bar",
        "barmode": "group",
        "ydata": {
          "value": {
            "label": "weight",
            "mode": "lines",
            "marker": {
              "line": {
                "width": 0
              },
              "color": "#c50e0e"
            }
          }
        },
        "xdata": "symbol",
        "showlegend": false,
        "outType": "graph",
        "xaxis": {
          "title": "stock symbol",
          "showgrid": false,
          "zeroline": false,
          "showticklabels": true
        },
        "yaxis": {
          "title": "weight",
          "showgrid": true,
          "zeroline": true,
          "showticklabels": true
        }
      },
      "height": 700
    }
  }
}

Map

Configuration

Example: Model Transport, parameter shipment quantities in cases

With regard to map-based graphics, a variety of possible visualizations are conceivable. In MIRO configuration mode, the following aspects can be realized on a map:

  • Markers
    Markers can be set. For this the position of the markers must be available in the data as longitude and latitude data. Marker labels can be set dynamically, i.e. based on data. In the following example, the marker label is set with Market: [j]. If you now click on a marker on the resulting map, the label will not be Market: [j]. Instead, the [j] is replaced with the city belonging to the data (here: Chicago).
    The data of this configuration is the following:
    The second column of the symbol schedule contains the markets j. The columns lngM and latM were selected as longitude and latitude data. For a given longitude-latitude combination, the corresponding element from the second column is now displayed as label.
  • Flows
    Between different points on the map flows can be displayed. The thickness of the arrows can depend on the flow data. The data overview from the previous picture also helps here for the understanding: In the example below the flows have been configured to point from lngP/latP to lngM/latM. The thickness depends on the data column quantities.
  • Charts
    You can display small charts on your map. In the following example, a pie chart is displayed for each of the 16 German states.
    The graph is based on the data below:
    The position of each pie chart results from a lngitude/latitude combination. The data of the pie chart for each state were determined in configuration mode with Renewable and Fossil.
    You also have the option of defining a time component in the charts. The data of the example covers a period of one year with a resolution of one hour. For the configuration, the corresponding 1st column was defined as the time dimension (starting with t001. Thus, for each time period, a separate pie chart can be displayed for each state (see play button at the bottom right of the map).

    Note that all data of your entire graphic must come from the same GAMS symbol. If you want to have more sophisticated graphics, it may be useful to combine data from different symbols into one symbol. Or you can use our R API and create your own graphics.

Layers
For map visualizations it can be helpful to display different layers, e.g. with markers. In configuration mode you can specify such layers. In the resulting graphic you can select the different layers and display only the corresponding markers.

JSON example


{
  "dataRendering": {
    "schedule": {
      "outType": "graph",
      "graph": {
        "title": "shipment quantities in cases",
        "tool": "leaflet",
        "layersControl": {
          "options": {
            "collapsed": true
          },
          "position": "topright",
          "overlayGroups": ["Canning plants", "Markets"]
        },
        "markers": {
          "1": {
            "lng": "lngp",
            "lat": "latp",
            "labelOptions": {
              "textsize": "12px",
              "permanent": false
            },
            "label": "Canning plant [i]",
            "group": "Canning plants"
          },
          "2": {
            "lng": "lngm",
            "lat": "latm",
            "labelOptions": {
              "textsize": "12px",
              "permanent": false
            },
            "label": "Market [j]",
            "group": "Markets"
          }
        },
        "flows": {
          "1": {
            "lng0": "lngm",
            "lat0": "latm",
            "lng1": "lngp",
            "lat1": "latp",
            "flow": "lngp",
            "color": "#0000ff",
            "minThickness": 1,
            "maxThickness": "8"
          }
        },
        "minicharts": {}
      },
      "height": 700
    }
  }
}

Time series diagram

Configuration

As the name of the graphic type suggests, the graphics configured here are based on time series. An important property of such time series is the format of the data. You should make sure that the dataset, which should represent the time component of your graphic, is available in a date format or at least can be formatted into such a format. A suitable syntax for a date would be yyyy-mm-dd, e.g. 2014-07-13.

Example: Model Pickstock, parameter dow jones vs. index fund

    "dowvsindex": {
      "outType": "graph",
      "graph": {
        "title": "dow jones vs. index fund",

        "xdata": "date",
        "ydata": {
          "dj": {
            "label": "dow jones",
            "stemPlot": false,

            "fillGraph": false,
            "drawPoints": false,
            "pointShape": "dot",
            "pointSize": 2
          },
          "index fund": {
            "label": "index fund",
            "stemPlot": false,
            "stepPlot": false,
            "fillGraph": false,
            "drawPoints": false,
            "pointShape": "dot",
            "pointSize": 2
          }
        },
        "dyOptions": {
          "includeZero": false,
          "logscale": false,
          "drawGrid": true,
          "stepPlot": false,
          "stemPlot": false,
          "fillGraph": false,
          "fillAlpha": 0.15,
          "drawPoints": false,

          "pointSize": 2
        },
        "dyHighlight": {

Pivot table

Configuration

The live preview of the pivot table already allows you to move and change the elements directly in the graphic. This can lead to confusion, because the actual configuration of the graphic has to be done on the left side as with all other graphics. Only the options set there are saved.

Example: Model Transport, parameter shipment quantities in cases

JSON example


{
  "dataRendering": {
    "schedule": {
      "outType": "pivot",
      "pivottable": {
        "aggregatorName": "Sum",
        "rendererName": "Heatmap",
        "locale": "en",
        "subtotals": false,
        "rows": "i",
        "cols": "j",
        "vals": "quantities"
      },
      "height": 700
    }
  }
}

Gantt chart

Configuration

The GAMS symbol, whose data should be visualized as a Gantt chart, must contain the following:

  • Start
    Each item must have a start variable. The format of the underlying data has to be in a date format, e.g. yyyy-mm-dd hh:mm:ss (hh:mm:ss is not required).
  • Content
    Each item must have a content variable which is then shown as a label.

The following example also contains:

  • End
    If the item is not a single point in time, but a range, you can provide an end variable.
  • Title
    A title is shown when you hover an item.
  • Group
    Items of the same groups are grouped together when specifying a group variable.

Note: The columns in the table do not have to be named start, end, content etc. Only the mapping you define in the configuation mode is relevant.

More inforation on the tool we use for gantt charts can be found here.

JSON example


{
  "dataRendering": {
    "gantt": {
      "outType": "graph",
      "graph": {
        "title": "asdasd",
        "tool": "timevis",
        "showZoom": true,
        "fit": true,
        "zoomFactor": 0.5,
        "series": {
          "1": {
            "content": "content",
            "start": "start",
            "type": "range",
            "end": "end",
            "title": "content",
            "group": "group",
            "groupTitle": "content"
          }
        },
        "editable": true,
        "multiselect": false,
        "showCurrentTime": false
      },
      "height": 700
    }
  }
}
