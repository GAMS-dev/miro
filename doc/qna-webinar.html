Q&A Webinar

Below you will find the questions and answers that emerged during our webinars on 20 and 21 February.
This page will be taken down in the near future!
Long-term relevant questions will be included in the FAQ section of the GAMS MIRO documentation.

General

Is there a version for BETA testing that I can try out?

Yes. An early BETA version of GAMS MIRO is available. If you would like to give it a try, please contact miro@gams.com.

When will GAMS MIRO 1.0 be released?

A final release is not yet scheduled (our plans for a first official release is summer 2019).

Is GAMS MIRO part of the regular GAMS distribution?

During the beta phase, a special GAMS distribution is available which contains a regular GAMS distribution (26.1) plus everything you need to run GAMS MIRO. For the official release, we are not clear yet in which form GAMS MIRO will be packaged and delivered.

Is there any kind of conference material that can be used for lectures/presentations?

Besides the webinar recording, the official documentation of GAMS MIRO is available at https://gams.com/miro

Technical

Can I use GAMS MIRO on MacOS?

Yes. GAMS MIRO can be used on Windows, MacOS, and Linux.

Can I set GAMS MIRO up on a server, where multiple users can solve GAMS models?

Yes. Besides the presented desktop version we also develop an enterprise version, which will run on a server and will offer features like multi-application management, user access management, load balancing, rolling updates, etc. The server version is based on Docker.

Can I use all solvers which I have a license for with GAMS MIRO?

Yes, all solvers with a valid license can be used with GAMS MIRO.

Will there be limits to the run time or the model size?

No, as long as a valid GAMS license exists, there are no limitations regarding the run time or size of a model.

Can I use GAMS MIRO without a GAMS model but work with data only?

Yes. However, it should be noted that GAMS MIRO is very closely woven with GAMS. In order to be able to use GAMS MIRO without GAMS, adjustments to the code of GAMS MIRO are required and the automatically written configuration files need to be maintained by hand.

What are the software requirements for 'customers', e.g., do they require full versions of the GAMS software?

The current GAMS MIRO desktop version requires a full GAMS installation including a proper GAMS license. If the need arises to decouple GAMS and GAMS MIRO for a desktop installation this can be accomplished (please contact us to discuss the requirements in detail) but our suggested way of deploying GAMS MIRO will be on a separate server. In this case, there are no requirements for the client machine.

Does GAMS MIRO work with Jupyter Notebook?

GAMS MIRO has been developed completely independent of the Jupyter Notebook interface. The Jupyter Notebook example from the webinar showed how an external analysis could look like. The scenario data was exported from GAMS MIRO (csv format) and used for a Jupyter Notebook analysis (see the example here). As mentioned during the webinar, we want to follow an approach that enables us to integrate such externally performed analyses into GAMS MIRO in the long term and thus use them as internal analysis.

Features

Do I need Excel installed when running GAMS MIRO?

No. The spreadsheet created when starting a GAMS MIRO app is generated with the help of internal tools. Excel is not necessary for this.

Are there any limitations when reading/writing an Excel file?

We are not aware of any limitations. The Excel workbook for important needs to follow certain rules, e.g. the sheet names need to correspond to the internal symbols names used by GAMS MIRO. But if you start with automatically generated Excel workbook you can adjust and improve this.

Will there be limits to the size of the output files that can be generated?

We are not aware of any limitations.

Are there any limitations when saving a ZIP file for external analysis?

We are not aware of any limitations. Of course, generating a very large ZIP file may take more time than generating a small one.

Did you find it difficult to upload and download large files using this system?

Downloading files is usually very fast. Uploading a ZIP file with scenario data can take a little more time, as the files are first validated internally and then saved in the database. Here the size of the zip file and especially the number of different files has the biggest impact on the time.

Can the data export be customized so that the export format is GDX (instead of CSV)?

Currently, CSV is the only format supported.

What does the “[MIRO:table]” mean in the pickstock model output parameters?

By including this keyword in the explanatory text of a parameter, each element of the last index set of this parameter will be shown in a separate table column in GAMS MIRO. To learn more about this, see here.

How can I start the MIRO Hypercube mode?

How to get to the Hyperube mode:
  • When you are in the base mode, you can simply click the “Switch to Hypercube mode” button in the header bar and you will automatically be redirected.
  • Alternatively, you can start the Hypercube mode directly via the switch: miromode=1 (in addition to miro=launch) from within GAMS Studio or the command line.

What happens with GAMS MIRO when using input data which results in infeasible or unbounded results?

In GAMS MIRO both the log and the lst file of the current GAMS model run are displayed. Any information about infeasibilities and unboundness is also visible here. Further checks on infeasibilities and unboundness are not performed. If a model is infeasible (or unbounded) this still leads to a normal completion of the GAMS job (unless you check the model status and trigger an abort based on an unexpected model status). Normal completion of GAMS is interpreted by GAMS MIRO as an error-free run. The view switches to the output section and will show the content of your output symbols (even if they are empty or show the infeasible solution provided by some first phase algorithm). If the GAMS model is aborted for any reason, this will also result in an abort in GAMS MIRO. A corresponding message is displayed in GAMS MIRO and no output is displayed.

Is it suitable to analyze geobased data?

So far, there is no specific geodata analysis tool available in GAMS MIRO. However, in the transport and transport_live demo models of the beta version, for example, geodata is used to set certain markers on a world map. This graphic was created using the R API for GAMS MIRO. This interface makes it possible to use the full range of visualization possibilities available in the R language. More on the topic of custom renderers can be found here.
If specific analyses are to be performed, scenario data generated with GAMS MIRO can be exported and analyzed externally.

Is it possible to have input validation functions in GAMS MIRO so it tells users if the input to the model is missing or falsely configured (before running models)?

The data layout and structures are given by the GAMS MIRO application and cannot be changed. The actual data might still lead to problems inside the GAMS model (including problems that occur during GAMS compilation like domain violations and logical problems during model execution). If case GAMS does not terminate normally, GAMS MIRO will let the user know and details about why are available in the log and listing file available in the GAMS MIRO interaction screen. Currently, checks such as range checks on input fields are not implemented. However, this can be easily extended as the underlying tools we use do support such input validation. If you are interested to know more about this, contact us at miro@gams.com.

Development

What language or toolset was used to implement GAMS MIRO?

MIRO is essentially based on the R language. Here we use in particular the Shiny package (see "https://shiny.rstudio.com/").

Pricing

How much do I need to pay for GAMS MIRO?

GAMS MIRO will be commercially supported. Exact prices have not been determined yet. However, most likely the desktop version of GAMS MIRO (without enterprise /server features) will be included in the GAMS/BASE package free of charge for our academic customers. We are undecided about commercial pricing for the desktop version, but this will be very much affordable (probably similar to the price of other GAMS components). There will be many commercial options for the Enterprise/server version including perpetual license, lease licenses, subscription, and use based. Also, the type of orchestration will play an important role in the consideration of the pricing. If you have a particular situation in mind, please contact us to discuss this in detail.

Is there going to be a student version for GAMS MIRO?

We are still somewhat unclear about the exact commercial pricing details of GAMS MIRO for our academic customers (see General questions) but most likely GAMS MIRO will be included in the GAMS/BASE module free of charge. We also will most likely provide some demo version of GAMS MIRO that works with a GAMS demo license.

Support

Who should I contact in case I have problems or remarks?

You can use any of the channels you also have if you face problems with the core GAMS system. However, the best way is via our forum. This way, other users who might have similar questions can profit as well from the answers you receive. Alternatively you can also contact us by mail at miro@gams.com
