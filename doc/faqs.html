FAQs

Last updated: October 12th, 2018

General

What GAMS version is required to run the WebUI?

In order to automatically generate the required configuration files via the / as well as / tags, GAMS version 25.1.2 or later is required.

Which operating systems are supported?

The local version of the WebUI runs on Windows, MacOS as well as Linux.

Does all the data for my model need to come from the WebUI?

No. You can read external data from any data sourcesupported by GAMS just as you did before. For example, you can place an Excel file in your model folder and import the data from this file into GAMS using the GDXXRW utility.
Note however, that the WebUI is running your GAMS model from a temporary working directory. This means that any $call or execute commands need to be executed with absolute paths instead of relative ones. The GAMS WebUI is providing you the path to your model via idir2. This means, in order to import e.g. an Excel file called test.xlsx, GDXXRW needs to be called like this: $call gdxxrw i=%gams.idir2%test.xlsx.

What languages are supported?

Currently the GAMS WebUI ships with language files for English and German. If you would like to add another language, you are more than happy to add more languages to this list. For more information about how to generate language files see the section on Language Files

Can I contribute to improve the GAMS WebUI?

Yes, absolutely! If you find bugs or have ideas about how to improve the WebUI, you are more than welcome to share those with us! Additionally, you can contribute by supplying new language files or custom renderers! We also welcome any prototypes/PoCs which you think that can help others to get started, to improve and/or to come up with new ideas of what can be done with the GAMS WebUI and optimization in general. In any case, please contact support@gams.com!

Features

How can I customize my WebUI?

Graphics, tables as well as other customization options are specified via the config.json file inside the directory: <modeldirecory>/config.json. In order to generate the config.json file you can use our Configuration Generator.

Do I have visualization options other than those shipping with the GAMS WebUI?

Yes. You can write your own Custom Renderers in R.

Can I also write my own custom functions to generate input data?

No. This is currently not supported, but might be added in later development stages depending on the demand for this feature.
