    Release notes

    GAMS MIRO 2.11

    (xxxx-xx-xx)

    New features

    • Add new optional scenario comparison module dashboard Compare. Like the dashboard renderer the dashboard scenario comparison is an interactive tool that can display multiple charts and tables but for mutliple scenarios at once.

    Minor new features and improvements

    • Dashboard renderer:
      • Add option to change the labels order in charts/tables.
      • Add option to hide the legend in charts.
      • Add option to show an alternative heatmap with different colors for positive and negative values.
      • Show comma as thousands separator in value boxes.
      • Support displaying the level of scalar variables or equations in value boxes.
    • New options in MIRO Pivot and dashboard renderer:
      • Support separate activation/deactivation of row/column summary.
      • Add option to configure filter dimensions as single-choice drop down menus.
      • Add option to specify the font size for charts.
      • Add support for a second x/y-axis.
      • Add option to select the second chart type in a combo chart.
      • It is now possible to configure the stacking behavior of data series in combo charts (as part of a stacked chart), enabling the creation of grouped stacked bar charts, stacked bar/line charts, and more.
      • Add option to draw a line chart as stepped line chart.
      • Add option to set a dash pattern for individual series of a line chart.
      • Add option to set a border width for individual series.
      • Add option to set a static minimum/maximum value for axes.
      • Add option to show/hide data points in a chart.
      • Add option to show/hide the grid in a chart.
      • Add option to specify the fill opacity for stacked and regular area charts.
      • Add option to stack positive and negative values together for stacked charts.
    • MIRO Pivot:
      • The view dialog has been restructured.
      • The settings dialog has been restructured.
      • Support setting title for table and heatmap renderers.
      • Add option to activate/deactivate fixed table columns to the settings menu.
      • The enableHideEmptyCols option has been deprecated. The functionality to show/hide empty table columns is now always accessible via the settings menu.
      • The CSV and PNG download buttons have been consolidated into a dropdown menu.
    • Pivot compare mode: Add support for setting default values for several configuration options that are also available in the regular Pivot tool.
    • Configuration Mode: When activating checkbox to load scenario on app startup, the scenario name is now "default" by default.
    • Improve error messages if the validation of the JSON configuration file fails.
    • Add GAMSPy example model: tsp.
    • Add option to parse standard log (stdout) for MIRO log syntax. This is particularly useful for GAMSPy-MIRO models, where it is easier to control the stdout of the process.

    Bug fixes

    • MIRO Pivot: Fix issue where removing title from chart did not work properly.
    • Pivot compare mode: Fix issue where removing all dimensions from rows could cause a problem in certain situations.
    • Update Help menu link to point to the new GAMS forum.
    • Fix issue where where clicking the "Solve" button could cause MIRO with GAMS Engine backend to crash in certain situations.
    • Fixes issue where updating dependent value(s) of slider resulted in broken slider with only NaN choices.
    • Fixes issue where updating dependent choices of dropdown resulted in selected value to be reset.
    • Fix issue where imported output datasets were not counted in the total number of imported sheets displayed in the alert message during local file imports.
    • Fix issue issue where choosing no datasets during manual dataset selection for local file imports incorrectly resulted in all datasets being imported.
    • Fix issue where adding GAMSPy apps to MIRO Desktop did not work in certain situations.
    • Fix crash when opening dialog to download temporary files when number of temporary files was large (>500).
    • Fix issue where rendering a Chart.js chart with an invalid configuration would break the canvas, preventing it from being reused for subsequent charts with valid configurations.
    • MIRO Server: Fix issue where app logos did not load when server.servlet.context-path was set to anything but /.
    • MIRO Pivot: Fix issue where incorrect labels were displayed in a chart in certain situations.
    • Using a fixed color mode (light or dark) did not work properly for some components (e.g. dashboard).

      • Add option to mix charting types (e.g. bar and line chart) as part of a view. Depending on the base chart type, one can now add additional bar/line series to the chart.
