Configuration Generator

Introduction

In order for you to be able to customize MIRO without having to write JSON code, we have developed this Configuration Generator. Here you can customize MIRO and charts. At the end you can download the JSON-file that needs to be placed in the folder <modeldirecory>/conf/config.json.

Note:

Like MIRO itself, the Configuration Generator is still in its early development stages. It is possible that a JSON configuration is generated, which does not fully fit into the required schema. In such a case we would be happy to hear from you.

Please also note that the Configuration Generator is currently only supported on Firefox and Chrome.

If you want to take a look at the schema used to validate the config.json file, you can do this here:

config.json - validation schema


When you want to use the Configuration Generator, the workflow can be the following:

  1. Click on Upload your GMSIO_config.json file
  2. Navigate to the directory of your model and upload the <modeldirectory>\conf\GMSIO_config.json file.
  3. Choose whether to generate a new config.json (recommended) or to merge with an existing one.
  4. The following screen appears:
  5. Now click through the menus and make the desired configurations.
  6. Click on next at the bottom of each page to go to the next page.
  7. Finally click on download JSON.
  8. The created config.json file or parts of it can now be copied into the existing file.
  9. The next time MIRO is started, the new config.json file is validated with the stored schema.
